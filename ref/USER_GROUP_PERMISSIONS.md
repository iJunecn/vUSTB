# 用户组默认权限整理

基于代码中的权限控制逻辑，整理超级管理员（super_admin）、管理员（admin）、教师（teacher）、服务器管理员（server_manager）、用户（user）五者的默认权限差异。

> 数据来源时间：2026-06-07
> 相关代码：`app/deps.py`、`app/utils/user_groups.py`、`app/routers/admin.py`、`app/routers/print_booking.py`、`app/routers/mc_servers.py`、`app/routers/article.py`

## 权限依赖函数

| 函数 | 允许的用户组 | 用途 |
|------|------------|------|
| `get_current_user` | 所有已登录用户 | 基础鉴权 |
| `get_current_admin` | super_admin, admin | 管理后台核心权限（用户管理、系统设置等） |
| `get_current_super_admin` | super_admin | 最高权限操作 |
| `get_current_server_content_manager` | super_admin, admin, server_manager | 服务器内容管理（材质、角色、动态、服务器列表） |
| `_can_manage_printer` | super_admin, admin, teacher | 打印系统管理权限 |

## 关键权限差异总结

### super_admin 独有权限
- 设置用户组为 super_admin / admin
- 查看/设置用户积分
- 创建 admin 邀请码

### admin vs server_manager
- **admin** 可访问完整管理后台（用户管理、系统设置、材质/角色/白名单管理、OAuth、Fallback）
- **server_manager** 仅可管理服务器列表、材质、角色、动态，和创建邀请码（普通用户/教师）
- **server_manager** 不可访问用户管理、打印管理、系统设置、OAuth、Fallback 等端点

### teacher vs server_manager
- **teacher** 权限集中在打印系统管理 + 创建邀请码（普通用户/教师）
- **server_manager** 权限集中在服务器内容管理（服务器、材质、角色、动态）+ 创建邀请码（普通用户/教师）
- 两者权限范围完全不同，互不重叠
- teacher 不能创建 server_manager 邀请码；server_manager 不能创建 teacher 邀请码
- **teacher** 不可访问 `admin.py` 的 `get_current_admin` 端点
- **server_manager** 不可访问 `admin.py` 的 `get_current_admin` 端点，但可访问 `get_current_server_content_manager` 端点

### teacher 定位
teacher 角色的 `is_admin = False`，不属于管理员组。其权限集中在打印系统管理，是"打印管理员"角色，而非系统管理员。

### server_manager 定位
server_manager 角色的 `is_admin = False`，不属于管理员组。其权限集中在服务器内容管理（服务器列表、材质、角色、动态），是"服务器管理员"角色，而非系统管理员。
